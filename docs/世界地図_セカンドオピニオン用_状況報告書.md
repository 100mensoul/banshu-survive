## 2. プロジェクト背景と世界地図の位置づけ

### 2.1 播州サバイブとは

READMEより:

> A personal storytelling project documenting my outsider perspective on my wife's grassroots revitalization journey in Banshu, Japan.

世界地図は、エピソード・ひめじん・相関図・ニュース瓦などと並ぶ**サイト内の一機能**であり、  
「ヒメモリの地」「コヌイの路」といった**物語上の地名**を、実際の播州の地形感覚と結びつける媒体です。

### 2.2 世界地図でやりたいこと（作者の意図）

- **地形を自分で sculpt する**（川を掘る、堤防を盛る、道路を引く）
- **スケール感を体験する**（人1.7m・車4.5mの凡例、グリッド≈50m感覚）
- **物語のスポットを地図上に置く**（slug付きマーカー）
- **エリア（色分け領域）を塗る**（約束の地、緑の子午線など）
- **複数の「世界」スケールを切り替える**（詳細1km圏 ↔ 姫路広域 ↔ 播磨全域）
- **最近の試み:** 幹線道路（播但連絡道路のような高架）を**3Dで表現**し、立体ビューで「道が空を渡る」感覚を出す

### 2.3 なぜ3D編集を試してきたか

| 動機 | 説明 |
|------|------|
| 没入感 | 平面だけでは「天川のフォーク」「川と道路の立体交差」が伝わりにくい |
| トレース作業 | Google Maps スクショを下絵にして、**編集画面のみ**俯瞰トレース（公開ページには不要） |
| 幹線道路 | 案B（路面高さデータ）→ 案C（立体メッシュ）と段階的に高架表現を追加 |
| コヌイの路 | 4ゾーンに分割し、④フォークノード（道の駅・天川付近）を詳細編集の主戦場に |

**現時点の率直な評価:** 3Dは「方向性として面白い」が、**幹線の見た目・縁の品質・データ同期**で試行錯誤が続いており、  
「このまま深掘りすべきか」「平面＋シンプルなシンボルに戻すべきか」判断に迷っている段階です。

## 5. データモデル

### 5.1 地形（頂点ごと、N = (SEG+1)²）

| 配列 | 意味 |
|------|------|
| `heights` | 地形の高さ（ワールド単位） |
| `water` / `waterY` | 水マスク / 水面の絶対Y |
| `groundMat` | 地表素材 ID（土・草・コンクリ・アスファルト・**幹線**） |
| `highwayDeckH` | 幹線の路面高さ（0=地上、>0=高架データ） |
| `areaGrid` | エリア塗りのID |

### 5.2 その他

- **樹:** `{ id, x, z, h, w, s, fc, tc }`（正規化座標）
- **スポット:** slug・名前・正規化 x,z（`link_type` は常に `none` — 未実装）
- **エリア定義:** `{ id, slug, name, color }`

### 5.3 Supabase `map_world_layers`

```
world_id, layer_id, seg,
heights, water, water_y, ground_mat, highway_deck,
area_defs, area_grid, vegetation,
updated_at
PRIMARY KEY (world_id, layer_id)
```

**layer_id の実例:**

- `hirao` — 旧・単一コヌイ層（フォークノードデータがここに残っている可能性）
- `hirao-zone-1` … `hirao-zone-4` — ゾーン分割後
- `hirao-overview` — 俯瞰平面

### 5.4 localStorage キー

| キー | 内容 |
|------|------|
| `banshu_world_map_v3_{worldId}` | 地形＋スポット等 |
| `banshu_world_map_v3_{worldId}_{zoneId}` | ゾーン別 |
| `banshu_world_map_v3_konui-michi` | **レガシー**（zone-4へ移行ロジックあり） |
| `banshu_world_map_road_styles_v1` | 道路色・幹線色・橋桁色・路面高・縁取り |
| `banshu_world_map_zone_trace_v1_{worldId}_{zoneId}` | トレース用スクショ（**クラウド非同期**） |

### 5.5 保存・読み込みの優先順位（要約）

1. Supabase から取得（カラム欠損時は段階的フォールバック）
2. 編集画面のみ: ローカルの `ground_mat` / `water_y` / `highway_deck` / 樹をマージ
3. `local.savedAt > remote.updatedAt` なら **ローカル優先**
4. zone-4 は `hirao-zone-4` が無い場合 **`hirao` 層にフォールバック**

## 7. 幹線道路・高架3D実験 — 経緯と現状

### 7.1 設計の段階

| 段階 | 内容 |
|------|------|
| 案A（当初） | 幹線も地形と一体（高さを盛る）→ 段差・浮遊ブロック問題 |
| **案B** | `highwayDeckH[]` に路面高さを記録、地形は盛らない |
| **案C** | 立体ビューで別メッシュ（デッキ＋橋桁）、平面は `groundMat` 色のみ |

### 7.2 現行定数（`world-map-editor.js`）

```
HIGHWAY_DECK_DEFAULT = 10m
HIGHWAY_DECK_THICKNESS_M = 1.5
HIGHWAY_PILLAR_SPACING_M = 28  （橋桁配置間隔。橋脚メッシュは未実装）
HIGHWAY_GIRDER_SLICE_M = 4
HIGHWAY_GIRDER_WIDTH_RATIO = 0.5
HIGHWAY_PLANK_WIDTH_M = 1.0
HIGHWAY_PLANK_GAP_M = 0.1
```

### 7.3 メッシュ生成パイプライン（`rebuildHighwayViaduct`）

1. 高架判定: `groundMat === 幹線` かつ `highwayDeckH` が閾値以上 かつ 地形より十分高い
2. 進行方向: 近傍セルの共分散（PCA風）＋ **平滑化角度**
3. **中心線セルのみ** に板張り路面（本体スラブ＋細長い板＋暗い溝）
4. 道幅は **ブラシ設定の固定幅**（セル実測幅ではなく）＋少し余白
5. 橋桁: 中心線上、28m間隔、幅は現地幅の50%、**道路に直角**（`rotY = theta`）
6. **橋脚（柱）は未実装** — UI文言に「橋脚」とあるが実際は橋桁のみ

### 7.4 試行錯誤で露呈したビジュアル問題

| 問題 | 原因の推定 | 試した対策 |
|------|------------|------------|
| お風呂マット状 | セル単位の正方形板＋溝のグリッド | 板張り（進行方向に細長い板）、厚み1.5m |
| 橋桁が斜め・はみ出る | グリッド軸に沿った回転、幅が道幅超過 | PCA角度、現地幅計測、中心線のみ、90°回転 |
| ウッドデッキに見える | 薄い板の継ぎ目 | 厚み増・回転・一体スラブ下敷き |
| **縁のギザギザ** | 斜め道路のマス目境界＋セルごと幅変動 | 中心線のみ・固定道幅・長い板の重ね |

**未解決感:** グリッドベースの根本限界は残る。完全な滑らかな縁には「中心線スイープによる連続メッシュ」や「テクスチャ平面」が必要かもしれない。

### 7.5 スタイル

- 幹線色・橋桁色: カラーピッカー（`road_styles` → localStorage のみ、クラウド非同期）
- 平面縁取り: チェックボックス（立体には影響しない）

## 8. 既知のバグ・データ問題

### 8.1 フォークノード（zone-4）データ

- ゾーン分割前の編集は `hirao` 層 / `banshu_world_map_v3_konui-michi` に存在
- `hirao-zone-4` 行が Supabase に無いと、読み込み時にフォールバックが必要
- **修正済み:** `fetchMapLayerRow()` で zone-4 は `hirao` にフォールバック
- **運用上:** 編集画面で④を開いて **Supabase保存** しないと公開に最新が反映されない可能性

### 8.2 スキーマ・同期のずれ

- `water_y`, `ground_mat`, `highway_deck`, `vegetation` が DB に無い環境では **ローカルのみ** 警告
- `mergeMatWaterYFromLocal()` がクラウド読込後にローカル幹線データを再マージ（幹線消失防止）
- `map_spots.world_id` の SQL デフォルト `banshu-main` ≠ 実行時の `konui-michi` 等

### 8.3 その他

- **リセット** はローカルのみクリア、Supabase 行は削除しない
- 下絵トレース・道路スタイルは **クラウドに上がらない**
- Undo の等価比較が地形中心で、樹・スポットが完全でない可能性
- 公開のコヌイゾーン詳細は **立体のみ**（平面切替不可）
