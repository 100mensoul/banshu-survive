# マスター引き継ぎ（banshu-survive）

## このファイルの役割：
- リポジトリ全体の方針・運用・開発の始め方だけを書く。
## ルール
### その日の作業の細部は書かない
- 細部は同フォルダ内の日付付きログを作成し、日付ごとに作成する
### マスターと、すべての日付付きログを確認してから作業する
- 日付付きログを最新版のみでなくすべて確認することで、見通しをふせぐ

## 最終更新情報
### 日付：2026-05-15 （最後に更新した日付）
### 内容：（簡単に2~３行で）
- **Cursor は、日付付きログの更新時に、必ずここも更新してください。**
- トップ **公式SNS**: Instagram 2アカウント横並び＋「Instagramを開く」ボタン表示中（確定デザイン・非表示の戻し方は日付ログ）。
- トップの DOM 順・ナビ・真実のストーリーは `2026-05-15-トップページ-真実のストーリーとナビ.md` を参照。

## なぜ「一枚の引き継ぎ」から変えたか

- 日々の出来事を **1行要約** に寄せると、後から **判断に必要な細部が落ちる**。
- 一方で、**すべてを1ファイルに無限追記**すると、古い記述の検索や当時の文脈の切り出しが難しくなる。
- そこで **マスター（全体・手順）** と **日付付きログ（その回の詳細・省略しない）** を分離し、開発開始時は **フォルダ単位で読む** ことで、全体把握と細部確認の両方を狙う。

## フォルダ構成と読み方（開発の最初にやること）

1. フォルダ **`docs/引き継ぎフォルダ/`** を Cursor で開く（または `@docs/引き継ぎフォルダ` でまとめて参照する）。
2. 必ず **`マスター引き継ぎ.md`（このファイル）** を先に読む。
3. 続けて、**日付の新しいログ**から必要な分だけ遡って読む（ファイル名は `YYYY-MM-DD-` で始める）。

**日付ログの命名例:** `2026-05-15-トップページ-真実のストーリーとナビ.md`

**日付ログに書くことの例:**

- 課題の背景（ユーザー視点・再現手順）
- 変更したファイルのパスと、変更の意図（行単位まで必要ならその旨）
- 取り得た案と採用／不採用の理由
- キャッシュバスター（`?v=`）を上げたか
- 次回の自分・他人への注意（保守で壊れやすい箇所）

**マスターに書くことの例:**

- リポジトリ・デプロイの入口
- 引き継ぎの運用ルール（このファイル）
- ローカル確認の **共通**手順（`http.server` など、ページに依存しない始め方）
- 別紙へのリンク（CMS など）

**ページや機能に固有の手順**（どのファイルを触るか、`?v=` のどこを上げたか、アニメの試し方など）は **マスターに書かず**、必ず該当する **`YYYY-MM-DD-*.md`** に書く。

## ローカルでサイトを開いて確認する（静的サイト）

このプロジェクトは **Node のビルド不要の静的 HTML** が中心です。`file://` で `index.html` を直接開くと、ブラウザやモジュールの制限で **挙動が変わることがある**ため、**簡易 HTTP サーバー**を使うのが確実です。

### 手順（ターミナル）

リポジトリのルート（`index.html` があるディレクトリ）に移動してから、次のどちらかを実行する。

**A. Python 3 が入っている場合（おすすめ）**

```bash
cd /Users/ootsukaumihei/Desktop/100mensoul-search/banshu-survive
python3 -m http.server 8765
```

ブラウザで次のいずれかを開く（**どちらも同じ**／トップは `index.html` が自動表示される）。

- **`http://127.0.0.1:8765/`**
- **`http://localhost:8765/`**

**B. Node が入っている場合**

```bash
cd /Users/ootsukaumihei/Desktop/100mensoul-search/banshu-survive
npx --yes serve -l 8765
```

表示された URL（多くは `http://localhost:8765` または `http://127.0.0.1:8765`）を開く。

### 止め方

ターミナルで `Ctrl + C`。

### CSS / JS を直したのに見た目が変わらないとき

ブラウザの **スーパーリロード**（キャッシュを無視して再読み込み）を試す。

- macOS Chrome: `Command + Shift + R`
- macOS Safari: メニュー「開発」→「キャッシュを空にする」など（環境による）

このリポジトリでは `index.html` などに **`?v=数字`** のクエリを付けてキャッシュを避ける運用がある。意味のある変更のあと **`?v=` を増やす** のも有効。**どのファイルのどのクエリを上げたか**は、該当作業の **日付ログ**に残す。
