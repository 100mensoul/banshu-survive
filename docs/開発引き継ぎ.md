# 開発引き継ぎ（入口）

このリポジトリの作業記録と手順は、**一枚にすべてを詰め込まない**運用に切り替えました。

## どこを読むか

1. **`docs/引き継ぎフォルダ/マスター引き継ぎ.md`**  
   全体方針・開発の始め方・ローカル確認の共通手順・リポジトリ概要。
2. **同じ `引き継ぎフォルダ` 内の `YYYY-MM-DD-*.md`**  
   その日の作業の **細部**（省略しないログ）。

Cursor では **`@docs/引き継ぎフォルダ`** のようにフォルダごと指定すると、マスターと日付ログをまとめて参照しやすいです。
